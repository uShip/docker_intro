Background

  • Docker started as an internal project at dotCloud (a platform as a service company)
  • Much of the tooling is what powered dotCloud's PaaS
  • Released open-source in March 2013
  • Since then it has recieved 25,600 github stars (aka the 20th most stared project on github)

What Docker is and is not:

  • It is a virtual environemnt

  • It is not a virtual machine
    • Docker is a linux only virtual environemnt tool.
    • It builds on libcontainer, which uses the cgroups functionality to enable creation and running of multiple isolated Linux virtual environments on a single control host.
    • Unlike a vm it does not create a virtual computer with a specific OS, processors, and hardware emulation.
    • Virtual environment is a vm-lite: it rides on the already existing kernel’s image of the underlying hardware, and only creates a ‘container’ in which to run your apps, and even recreate the OS if you want since the OS is also just another app running on the kernel.
    • As a result, unlike virtual machines, there is very little overhead involved with running docker.

    How it works

  • Written in Go

  • Uses helper scripts called (Dockerfiles) to build containers as lightweight machines.
